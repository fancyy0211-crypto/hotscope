# HotScope

HotScope 是一个面向内容运营的 AI 热点决策与选题工具。

核心目标：
帮助内容运营人员在复杂信息环境中，快速完成：
- 热点发现
- 价值判断
- 跟进策略选择
- 内容选题生成

---

## 🧠 产品核心能力

### 1. 热点聚合
聚焦微博热榜热点数据，统一结构展示并驱动后续决策流程。

### 2. 机会评估
对每个热点进行量化评分（如：热度、机会值），并提供可解释的评分构成。

### 3. 行业筛选
支持多行业过滤，所有推荐逻辑必须基于当前行业上下文。

### 4. 策略驱动选题生成
用户选择不同跟进策略（如：深挖痛点 / 对比产品 / 情绪共鸣），系统生成不同结构的内容选题。

### 5. AI 辅助决策
提供：
- 最佳机会推荐
- 行动建议（Action Advice）
- 多方案对比（Decision Support）

---

## ⚙️ 核心原则

HotScope 不是一个“展示工具”，而是一个**决策工具**。

必须保证：

- 用户选择 → 影响推荐
- 用户操作 → 改变结果
- 所有模块 → 有真实数据逻辑

---

## 🚫 禁止行为（非常重要）

系统中严禁出现：

- 假推荐（与筛选无关）
- 假筛选（UI变化但数据不变）
- 假策略（选择后不影响结果）
- 展示型数据（不会驱动逻辑）

## ⏰ 服务端定时简报（Vercel Cron）

项目已支持服务端定时发送（页面关闭后仍可发送）：

- 配置管理 API：`/api/digest-configs`
- 定时任务 API：`/api/cron-digest`
- 定时规则：`vercel.json`（每分钟触发一次）

### 必需环境变量（Vercel Project Settings）

- `EMAILJS_SERVICE_ID`
- `EMAILJS_TEMPLATE_ID`
- `EMAILJS_PUBLIC_KEY`
- `KV_REST_API_URL`
- `KV_REST_API_TOKEN`
- `CRON_SECRET`（可选，用于手动触发 cron 接口鉴权）

说明：
- 若未配置 KV，服务端会退回内存存储，重启后配置会丢失，不适合生产。
- 生产环境建议必须配置 KV，以保证定时任务读取到持久化配置。
